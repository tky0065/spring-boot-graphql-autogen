Fonctionnalités principales

GraphQL AutoGen simplifie votre développement avec des outils puissants et intuitifs

Zéro configuration

Ajoutez simplement la dépendance, annotez vos entités et c'est prêt ! Pas de boilerplate, pas de schémas à écrire.

Annotations intuitives

Des annotations simples pour marquer vos types, champs, requêtes et mutations GraphQL.

DataLoaders automatiques

Résolution automatique des problèmes N+1 avec génération de DataLoaders pour les relations entre entités.

Pagination avancée

Support de pagination Offset et Relay Cursor avec filtrage et tri dynamiques.

Documentation intégrée

Génération automatique de la documentation GraphQL à partir des JavaDocs et des descriptions d'annotations.

Outil CLI

Générez vos schémas GraphQL directement depuis le terminal, sans démarrer l'application.

Démarrez en 2 minutes

Ajoutez la dépendance, annotez vos classes, et c'est tout !

1. Ajouter la dépendance

Maven
Gradle
<dependency>
    <groupId>com.enokdev</groupId>
    <artifactId>graphql-autogen-spring-boot-starter</artifactId>
    <version>1.0.0-SNAPSHOT</version>
</dependency>
implementation 'com.enokdev:graphql-autogen-spring-boot-starter:1.0.0-SNAPSHOT'

2. Annoter votre entité

@Entity
@GraphQLType(name = "Product")
public class Product {

    @Id
    @GraphQLId
    private Long id;
    
    @GraphQLField(description = "Nom du produit")
    private String name;

    @GraphQLField(description = "Prix du produit")
    private Double price;

    // Getters & Setters
}

3. Et c'est tout !

Démarrez votre application Spring Boot et accédez à http://localhost:8080/graphiql pour commencer à utiliser votre API GraphQL :

query {
  products {
    id
    name
    price
  }
}

Utilisation du CLI

Pour générer des schémas GraphQL sans démarrer l'application :

# Installation
java -jar graphql-autogen-cli.jar --install

# Génération d'un schéma
graphql-autogen --scan-package com.example.model --output schema.graphql
